# Reglas y Políticas

1. **Modularidad**: Separar código en módulos reutilizables. Documentar y probar.
2. **Verificación continua**: Generar artefactos de pruebas unitarias y de integración.
3. **Accesibilidad y UX**: Estándares WCAG 2.1, contraste, navegación por teclado.
4. **Gestión de modelos**: Seleccionar el modelo más eficiente en coste para cada subtarea.
5. **Privacidad**: No exponer datos sensibles; anonimizar muestras.
6. **Iteración ágil**: Dividir historias de usuario en subtareas manejables.
